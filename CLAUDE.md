# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

**Prépa ATSEM** (prepa-atsem.fr) is a French-language preparation platform for the **concours d'ATSEM** (Agent Territorial Spécialisé des Écoles Maternelles) — the entrance exam for working as a specialized assistant in French nursery schools. The site targets candidates preparing the external, internal, and 3rd-way concours: aspiring ATSEM, holders of CAP AEPE, and people in career change.

The codebase is a fork of Prépa FPC (nursing school exam prep), now fully adapted to ATSEM.

## Tech Stack

- **Framework**: Next.js 16 with App Router (JavaScript, not TypeScript despite tsconfig)
- **Styling**: Tailwind CSS v4 (via PostCSS plugin)
- **Backend/Auth/DB**: Supabase (auth + Postgres + storage, with RLS enabled on every table)
- **AI**: Anthropic Claude (Haiku 4.5) for QCM generation and oral CV analysis
- **Payments**: Stripe (checkout, customer portal, webhook)
- **Email**: Resend for transactional email
- **Deployment**: Vercel

## Commands

- `npm run dev` — Start dev server

- `npm run lint` — Run ESLint

## Architecture

All pages are client components (`'use client'`) using plain `.js` files in the App Router (`app/` directory). There is no component library or shared component extraction — navigation bars, auth logic, sidebars and SVG icons are duplicated inline across pages. **This is intentional for fast iteration; do not refactor into shared components without explicit user request.**

### Brand & design system

- **Primary color**: `purple-700/800` (Tailwind). "ATSEM" in the brand name is always `text-purple-800`.
- **Font**: **Nunito** (Google Fonts, weights 400/600/700/800/900). Loaded inline via `<link>` on every public page, also set in `style={{ fontFamily: "'Nunito', system-ui, sans-serif" }}` on the root container.
- **Background halo**: Public pages share a 5-radial-gradient halo (purple/amber/pink/sky) on a `#faf8ff` base. The exact gradient string is duplicated inline — copy it verbatim from `app/page.js`, `app/auth/page.js`, etc. when adding a new public page:
  ```
  backgroundColor: '#faf8ff',
  backgroundImage: 'radial-gradient(ellipse at 0% 0%, rgba(139,92,246,0.18), transparent 60%), radial-gradient(ellipse at 100% 0%, rgba(251,191,36,0.13), transparent 60%), radial-gradient(ellipse at 50% -10%, rgba(236,72,153,0.10), transparent 55%), radial-gradient(ellipse at 0% 100%, rgba(14,165,233,0.10), transparent 60%), radial-gradient(ellipse at 100% 100%, rgba(236,72,153,0.14), transparent 60%)'
  ```
- **Logo**: inline SVG (two ATSEM silhouettes around a star) in `app/page.js`, `app/dashboard/page.js`, etc. — uses `currentColor` so its color matches its parent.
- **Card style**: `bg-white rounded-3xl shadow-2xl border border-slate-100 p-6 sm:p-10` is the canonical card on auth/forgot/reset pages.
- **Buttons**: primary CTA = `bg-slate-900 hover:bg-black text-white font-bold rounded-xl`. Disabled = `bg-slate-200 text-slate-400 cursor-not-allowed`.

### Responsive

- Mobile-first. Tested breakpoints: 375px (mobile), 768px (tablet), 1440px (desktop).
- Stats grids use `grid-cols-1 sm:grid-cols-3 md:grid-cols-5` patterns with `gap-4 sm:gap-6`.
- On mobile, items in stat strips are centered (`flex flex-col items-center text-center`).
- Padding scales: `p-4 sm:p-6` on cards, `py-3.5` on inputs, `py-4` on primary buttons.

### Key files

#### Pages (authenticated)
- `app/dashboard/page.js` — User dashboard, sidebar navigation, tabs for stats / history / profile / subscription. Single file containing all tab UIs.
- `app/examen/page.js` — Mock exam (45 min, 20 multi-answer questions) generated by Claude. The API supports inspiration from real annales. Server shuffles answer positions to avoid A/B bias.
- `app/annales/page.js` — Redirects to a random annale ID (no listing page).
- `app/annales/[id]/page.js` — Display one real annale (45 min QCM with multi-answer questions). Data stored in Supabase `annales` table.
- `app/specifique/page.js` — Thematic single-answer QCM (10 questions per category) generated by Claude. 6 categories: Institutionnel, Hygiène, Santé, Pédagogie, Relations pro, Calculs. Per-question validation flow with immediate explanation panel. Nav bar tints to category color via `--nav-color` / `--nav-tint` CSS vars.
- `app/oral/page.js` — Oral exam prep: uploads CV PDF, sends to Anthropic Claude API, generates personalized interview questions. RAF-driven arc loader (no CSS transition on `stroke-dashoffset` — caused visual lag).
- `app/admin/page.js` — Admin panel for blog article CRUD (restricted to a hardcoded admin email).

#### Pages (public)
- `app/page.js` — Landing page with inline interactive QCM teaser, FAQ, eligibility section, 3 voies d'accès, 5-stat hero strip.
- `app/qcm/page.js` — Public quiz teaser with hardcoded question bank from `lib/quizData.js`. Sample on the landing — NOT the same as `/specifique`.
- `app/blog/page.js` and `app/blog/[slug]/page.js` — Blog with articles stored in Supabase `articles` table. Article page is white background; index uses the halo.
- `app/tarifs/page.js` — Pricing. Both cards use `light` style, equal width (`1fr 1fr`), with "Examen blanc généré par IA" as 3rd feature and "Renouvellement automatique" as 5th.
- `app/calendrier/page.js` — Concours calendar. 5-step diamond timeline (`.cal-tl-dot { transform: rotate(45deg) }`), current step highlighted dynamically via `Date.now()` vs `endsAt`. Clickable France map SVG; region cards in dark style for active 2026. `REGION_LINKS` couples IDF↔CVL and PAC↔COR.
- `app/auth/page.js` — Login/signup. `/login` and `/signup` redirect here.
- `app/forgot-password/page.js`, `app/reset-password/page.js` — Password reset flow via Supabase. Reset enforces: 8+ chars, 1 uppercase, 1 number, 1 special char.
- `app/cgu/page.js`, `app/mentions-legales/page.js` — Legal.

#### API routes
- `app/api/specifique/route.js` — Generates a 10-question thematic QCM via Claude (action: `generer`, body: `{ categorie }`). JSON parser with comma-insertion repair and 3-retry loop.
- `app/api/maths/route.js` — Generates the 20-question mock exam. Action `generer_questions` injects 3 random annales × 3 questions as inspiration. Action `generer_correction` produces the answer key. Server-side answer shuffling.
- `app/api/oral/route.js` — Sends uploaded PDF to Anthropic Claude (Haiku 4.5), returns 10 personalized interview questions.
- `app/api/stripe/checkout/route.js`, `portal/route.js`, `webhook/route.js` — Stripe integration.
- `app/api/review/route.js`, `support/route.js`, `contact/route.js`, `emails/check/route.js` — Misc utilities.

#### Library
- `lib/supabase.js` — Single Supabase client (uses `NEXT_PUBLIC_SUPABASE_URL` and `NEXT_PUBLIC_SUPABASE_ANON_KEY`).
- `lib/quizData.js` — Hardcoded question bank for the public `/qcm` teaser only.
- `lib/rate-limit.js` — Simple in-memory rate limiter (per IP). Post-launch: replace with Upstash Redis for distributed rate limiting.
- `lib/daily-limit.js` — Per-user daily limits.
- `lib/email.js` — Email sending helper (Resend).
- `lib/prompts/qcm-thematique.js` — System + per-category prompts for `/specifique` QCM generation.
- `lib/prompts/examen.js` — System + prompts for the mock exam (`/examen`).
- `lib/prompts/base-oral.js`, `simulation-oral.js` — Oral prep prompts.

#### Database tables (Supabase) — RLS enabled on all
- `subscriptions` — Stripe subscription state (`status`, `current_period_end`). Writes blocked at DB level except via service role (Stripe webhook).
- `historique` — Per-user activity log: `type` ('Examen', 'Annale', 'Spécifique', 'Oral'), `label`, `note`, `note_max`, `nb_questions`, `duration_minutes`.
- `scores_annales` — Detailed annale results (per-question answers).
- `annales` — Real exam content: `region_nom`, `annee`, `cdg`, `duree_minutes`, `nb_questions`, `bareme`, `questions` (JSONB). Read-only for users.
- `articles` — Blog articles. Write restricted to admin email via RLS.

### Auth pattern

Every authenticated page follows the same pattern, repeated inline:

```js
useEffect(() => {
  supabase.auth.getSession().then(async ({ data: { session } }) => {
    if (!session) { window.location.href = '/auth'; return }
    setUser(session.user)
    const { data: sub } = await supabase.from('subscriptions')
      .select('status, current_period_end')
      .eq('user_id', session.user.id).eq('status', 'active').single()
    const hasSub = sub && new Date(sub.current_period_end) > new Date()
    if (hasSub) setIsPremium(true)
    const trialMs = 7 * 24 * 60 * 60 * 1000 - (Date.now() - new Date(session.user.created_at))
    if (!hasSub && trialMs <= 0) { setShowAccessBlock(true); setAuthLoading(false); return }
    setAuthLoading(false)
  })
}, [])
```

Free trial is 7 days from account creation; after that, a premium subscription is required to access exercises. **Note**: client-side gating only for now — server-side `checkPremium` (validate JWT + subscription in API routes) is on the post-launch roadmap.

### SEO

- Per-page `layout.js` files set the `title` to the bare subject name (e.g. `"Tarifs"`); the root layout's `title.template: "%s | Prépa ATSEM"` adds the suffix automatically. **Do not** include "| Prépa ATSEM" in per-page titles — it produces duplicates.
- JSON-LD structured data is inlined: `BreadcrumbList`, `Organization`, `FAQPage` (landing), `Event` (calendrier).
- Pre-launch deployment tasks: fix `robots.txt` (allow public, block `/dashboard`, `/admin`, `/auth`, `/api`); generate `sitemap.xml`.

## Language

All user-facing content is in French. Maintain French for UI text, error messages, button labels, and code comments. Use vouvoiement (formal "vous") with users.

## Conventions

- No TypeScript types — plain JS only.
- No shared component extraction. Inline duplication is preferred.
- Lucide-react for icons in authenticated pages; inline SVG paths on the landing/teaser pages.
- French Tailwind comments are fine (`{/* ===== POPUP INFO ===== */}`).
- Always save quiz results to `historique` with the appropriate `type`, so they show up in dashboard stats.
- **Tailwind purge gotcha**: never interpolate color names into class strings (e.g. `border-${color}-200`) — Tailwind purges them at build. Use a static class map literally referenced in source:
  ```js
  const cls = { border: 'border-purple-200/50', bg: 'bg-purple-100', text: 'text-purple-800' }
  ```
- **SVG paths**: avoid mixing absolute jumps (`M`, `H`, `V`) inside relative chains (`m`, `h`, `v`) — caused a buggy mic icon. Prefer Lucide's source paths verbatim.
- **Animated SVG arcs**: don't put a CSS `transition` on `stroke-dashoffset` — drive the value directly with `requestAnimationFrame` to keep loaders in sync.
