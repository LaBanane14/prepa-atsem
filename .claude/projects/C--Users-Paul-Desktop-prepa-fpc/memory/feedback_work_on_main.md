---
name: Toujours travailler sur main
description: Ne pas utiliser de worktree ou branche séparée — travailler directement sur la branche main
type: feedback
---

Toujours travailler directement sur la branche main, pas dans un worktree ni sur une branche séparée.

**Why:** L'utilisateur veut voir les changements immédiatement dans son dossier projet local. Travailler dans un worktree crée de la confusion car les fichiers n'apparaissent pas dans le répertoire principal.

**How to apply:** Ne jamais créer de worktree ni de branche feature. Commit et push directement sur main.
